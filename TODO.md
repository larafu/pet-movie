## 视频存储系统实现

### 优先级：中

当前使用 Replicate API 生成视频，但 API 仅保留生成视频 1 小时。需要实现自有存储方案，确保用户可长期访问其生成的视频。

#### 1. 选择并配置存储方案
- [ ] 集成 Cloudflare R2 作为主存储（成本低，零出口费用）
  - 存储成本：$0.015/GB/月
  - 出口流量：$0（免费）
  - 预计成本：1万用户约$75/月
- [ ] 配置 R2 存储桶和访问密钥
- [ ] 更新环境变量配置

**影响文件**：
- `src/extensions/storage/` - 存储服务集成
- `.env` - 添加 R2 配置

#### 2. 实现视频自动保存
- [ ] 在 Replicate 视频生成完成后自动下载
- [ ] 上传视频到 R2 存储
- [ ] 更新数据库中的视频 URL 为 R2 地址
- [ ] 实现重试机制（处理网络失败）

**影响文件**：
- `src/app/api/ai/generate/route.ts` - 生成完成回调
- `src/shared/models/ai_task.ts` - 添加存储逻辑

#### 3. 分层存储策略（优化成本）
- [ ] 实现存储策略：
  - 免费用户：保留 30 天或最多 3 个视频
  - 付费用户：无限期保留
- [ ] 添加定时任务清理过期视频
- [ ] 实现用户手动删除功能

**影响文件**：
- 需要创建 Cron Job 或定时任务
- `src/shared/models/ai_task.ts` - 添加删除逻辑

#### 4. 用户视频管理功能
- [ ] 实现"我的视频"页面（当前屏蔽中）
- [ ] 显示用户所有生成的视频
- [ ] 提供下载功能
- [ ] 提供删除功能

**影响文件**：

#### 5. 监控和告警
- [ ] 记录存储使用量

**技术参考**：
- Cloudflare R2 文档：https://developers.cloudflare.com/r2/
